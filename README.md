# VIRTUAL LIBRARY

Web application to manage a multimedia library. 


## Commit guidelines

The format of the commit messages should conform the [Conventional Commits 1.0](https://www.conventionalcommits.org/en/v1.0.0/) specification:

```
type: subject

body (optional)

footer (optional)
```

Usual commit types:

- feat - a new feature
- fix - a bug fix
- docs - changes in documentation
- style - everything related to styling
- refactor - code changes that neither fixes a bug or adds a feature
- test - everything related to testing
- chore - updating building tasks, package manager configs, etc.
